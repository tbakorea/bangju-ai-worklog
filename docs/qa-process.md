# Bangju AI Worklog QA Process

## 목적

디자인 수정 후 특정 화면, 권한, 기기폭 조합에서 UI가 깨지는 일을 줄이기 위한 자체 점검 절차입니다.

## 기본 실행

```bash
./scripts/qa-check.sh
```

이 검사는 다음 항목을 확인합니다.

- `app.js` 문법 오류
- HTML 중복 ID
- 메뉴 버튼과 실제 화면 섹션 연결
- HTML에서 참조하는 로컬 파일 존재 여부
- CEO 모드의 전 사업장 업무일지 레이아웃 보호 규칙
- 날짜 이동 UI 필수 요소
- 휴대폰에서는 CEO 전용 모드 유지

## 화면별 필수 시운전

수정 후 아래 조합을 확인합니다.

1. iPhone 폭
   - 피트니스 업무일지
   - 일반 업무일지
   - 오늘의 업무 확대/확인
   - 시간별일정 확대/확인

2. iPad 폭
   - CEO 모드 업무일지
   - 클래식 모드 업무일지
   - 전 사업장 업무일지
   - 날짜 이동과 달력

3. 컴퓨터 폭
   - 전 사업장 업무일지 가로 스크롤
   - 대표 경영페이지
   - 통합관제
   - 직원/노무/보고서

## 눈으로 보는 체크포인트

- 날짜가 `...`로 잘리지 않는가
- 제목이 세로로 깨지지 않는가
- 버튼 글자가 밖으로 넘치지 않는가
- CEO/클래식 전환 시 화면 폭이 의도대로 바뀌는가
- 휴대폰에서는 CEO/클래식 전환 버튼이 숨겨지는가
- 대표 계정에서는 가입승인과 전 사업장 업무일지가 보이는가
- 일반 직원에게는 허용된 메뉴만 보이는가

## 커밋 전 순서

```bash
./scripts/qa-check.sh
git diff --check
git status --short
```

세 가지가 깨끗하면 커밋하고 푸시합니다.
